Continuing the GalaPo project. Build backend APIs for the business owner dashboard.

TECH: Next.js 16 Route Handlers, Supabase, TypeScript

REQUIREMENTS:

1. /app/api/business/dashboard/stats/route.ts:
   - GET: Fetch dashboard stats for authenticated business owner
   - Returns:
     {
       total_listings: number,
       total_views_this_month: number,
       total_views_last_month: number,
       views_change_percent: number,
       total_clicks_this_month: number,
       total_clicks_last_month: number,
       clicks_change_percent: number,
       active_deals: number,
       active_subscriptions: number
     }
   - Query listing_analytics table for views/clicks
   - Group by current month vs previous month
   - Only for listings owned by this user
   - Requires authentication

2. /app/api/business/listings/route.ts:
   - GET: Fetch all listings owned by authenticated user
   - Include: primary_image, category_name, subcategory_name, status, current_plan
   - Include per-listing: views_this_month, clicks_this_month
   - Order by: created_at DESC
   - Pagination: ?page=1&limit=20
   - No status filter (show all: pending, approved, rejected)

3. /app/api/business/activity/route.ts:
   - GET: Fetch recent activity for authenticated user
   - Sources:
     - Listing status changes (approved, rejected)
     - Deals created/expired
     - Events created
     - Payments confirmed
     - Subscription changes
   - Combine and sort by date DESC
   - Limit: 10
   - Returns: [{ type, title, message, timestamp, listing_id, listing_name }]

4. /app/api/business/profile/route.ts:
   - GET: Fetch current user's profile
   - PUT: Update profile (full_name, phone, avatar_url, notification_preferences)
   - Validation: phone format, name not empty
   - Handle avatar upload URL (file uploaded separately to Supabase Storage)

5. /app/api/business/profile/password/route.ts:
   - PUT: Change password
   - Body: { current_password, new_password }
   - Verify current password first (re-authenticate with Supabase)
   - Update to new password
   - Return success/error

6. /app/api/business/profile/avatar/route.ts:
   - POST: Upload avatar image
   - Accept multipart form data
   - Upload to Supabase Storage "logos" bucket (path: avatars/[user_id])
   - Return public URL
   - Delete old avatar if exists

7. UPDATE /lib/types.ts:
   - Add DashboardStats type
   - Add ActivityItem type
   - Add NotificationPreferences type (JSONB shape)

8. Add notification_preferences JSONB column to profiles table (provide SQL):
   - Default: { email_listing_status: true, email_subscription_expiry: true, email_annual_check: true, email_payment: true }

Output all files with complete SQL and TypeScript code.
