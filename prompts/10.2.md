Continuing the GalaPo project. Build backend APIs for the super admin dashboard.

TECH: Next.js 16 Route Handlers, Supabase, TypeScript

REQUIREMENTS:

1. /app/api/admin/dashboard/stats/route.ts:
   - GET: Fetch all dashboard statistics
   - Requires super_admin role
   - Returns:
     {
       total_active_listings: number,
       listings_change: number (vs last month),
       pending_approvals: number,
       pending_payments: number,
       pending_claims: number,
       active_subscriptions: number,
       total_page_views_this_month: number,
       views_change: number (vs last month),
       total_business_owners: number,
       revenue_this_month: number,
       revenue_last_month: number,
       revenue_all_time: number,
       revenue_breakdown: {
         subscriptions: number,
         ad_placements: number,
         top_search: number,
         reactivation_fees: number
       },
       annual_checks_due_this_week: number,
       annual_checks_no_response: number
     }

2. /app/api/admin/dashboard/activity/route.ts:
   - GET: Fetch recent platform-wide activity
   - Sources: ALL tables (listings, payments, claims, subscriptions, registrations)
   - Combine into unified timeline
   - Query params: ?limit=20&page=1
   - Each activity: { type, title, description, user_name, timestamp, link }
   - Order by: timestamp DESC

3. /app/api/admin/dashboard/top-listings/route.ts:
   - GET: Top performing listings by page views
   - Query param: ?period=this_month (or last_month, all_time)
   - Returns top 10 with: listing_name, category, views, clicks, plan_type, owner_name

4. /app/api/admin/dashboard/pending/route.ts:
   - GET: Fetch all pending items for quick review
   - Returns:
     {
       pending_listings: [{ id, business_name, owner_name, submitted_at }] (limit 5),
       pending_payments: [{ id, owner_name, amount, plan, submitted_at }] (limit 5),
       pending_claims: [{ id, listing_name, claimant_name, submitted_at }] (limit 5)
     }

5. /app/api/admin/users/route.ts:
   - GET: List all business owner accounts
   - Query params: ?search=name&status=active&page=1&limit=20
   - Returns: profiles with listing count, subscription status
   - Sortable by: name, email, created_at, listing_count

6. /app/api/admin/users/[id]/route.ts:
   - GET: Single user profile with all their listings
   - PUT: Update user (toggle is_active, change role)
   - Cannot deactivate yourself (super admin self-protection)

7. /app/api/admin/claims/route.ts:
   - GET: List all claim requests
   - Query params: ?status=pending&page=1&limit=20
   - Returns: claim info + listing info + claimant info + proof document URL

8. /app/api/admin/claims/[id]/route.ts:
   - PUT: Approve or reject claim
   - Body: { action: "approve" | "reject", reason?: string }
   - On approve:
     1. Set listing.owner_id to claimant's user ID
     2. Set listing.status to 'approved'
     3. Set listing.claimed_at to now
     4. Set listing.is_pre_populated to false
     5. Create notification for business owner: "Your claim has been approved!"
     6. Send email with approval + premium listing benefits ad
   - On reject:
     1. Set listing.status back to 'approved' (remains public but unclaimed)
     2. Clear listing.claim_proof_url
     3. Create notification: "Your claim was not approved. Reason: [reason]"
     4. Send email with rejection reason

9. /app/api/admin/annual-checks/route.ts:
   - GET: List all annual checks
   - Query params: ?status=pending|no_response|confirmed&page=1
   - Returns: check info + listing info + owner info

10. /app/api/admin/annual-checks/trigger/route.ts:
    - POST: Manually trigger annual check for specific listing
    - Body: { listing_id }
    - Creates annual_checks record
    - Sends notification + email to business owner (if registered)

11. /app/api/admin/annual-checks/batch-trigger/route.ts:
    - POST: Auto-trigger annual checks for all listings older than 1 year since last_verified_at
    - Creates annual_checks records for each
    - Sends notifications + emails
    - Returns: { triggered_count: number }

12. /app/api/admin/annual-checks/[id]/route.ts:
    - PUT: Update annual check status
    - Body: { action: "confirm" | "deactivate" | "extend" }
    - On confirm: update listing.last_verified_at, set check status = 'confirmed'
    - On deactivate: set listing.is_active = false, set check status = 'deactivated', create reactivation_fee record
    - On extend: extend response_deadline by 7 more days

13. CREATE /lib/admin-helpers.ts:
    - requireAdmin(request): verify super_admin role, throw 403 if not
    - getAdminDashboardStats(): aggregate all stats
    - getPendingCounts(): quick count of pending items
    - formatAdminActivity(rawData): format into unified activity timeline

    - Runs daily at midnight PH time
    - Finds listings where last_verified_at < NOW() - INTERVAL '1 year'
    - AND no active annual_check exists for that listing
    - Creates annual_check records
