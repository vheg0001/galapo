Continuing the GalaPo project. Build backend APIs for admin listing management.

TECH: Next.js 16 Route Handlers, Supabase, TypeScript

REQUIREMENTS:

1. /app/api/admin/listings/route.ts:
   - GET: List all listings with comprehensive filtering
   - Query params:
     - status (pending|approved|rejected|draft|claimed_pending)
     - category (slug)
     - subcategory (slug)
     - barangay (slug)
     - plan (free|featured|premium)
     - has_owner (true|false)
     - is_active (true|false)
     - search (business_name, email, phone search)
     - date_from, date_to
     - sort_by (business_name|category|status|created_at|views)
     - sort_order (asc|desc)
     - page, limit
   - Returns: listings with owner name, category name, barangay name, views count, current plan
   - Includes counts per status for tabs
   - Requires super_admin role
   
   - POST: Admin create listing (same as business owner but with admin extras)
   - Body includes: status, is_featured, is_premium, is_active, owner_id (optional)
   - If no owner_id → is_pre_populated = true
   - Status defaults to 'approved'

2. /app/api/admin/listings/[id]/route.ts:
   - GET: Full listing detail with ALL related data
     - Listing fields
     - Owner profile
     - Images
     - Dynamic field values
     - Deals
     - Events
     - Current subscription
     - Payment history
     - Analytics summary (views, clicks totals + this month)
     - Claim info (if claimed_pending)
     - Admin notes
   - PUT: Update any listing field
     - Admin can change: status, is_featured, is_premium, is_active, owner_id, all content fields
     - No re-approval needed (admin IS the approver)
     - Log the change in admin activity
   - DELETE: Hard delete (with confirmation) or soft delete (is_active = false)
     - Also deletes related: images (from storage), field values, analytics
     - Or soft delete: just set is_active = false

3. /app/api/admin/listings/[id]/approve/route.ts:
   - POST: Approve a pending listing
   - Sets status = 'approved'
   - Sets last_verified_at = now
   - Creates notification for owner: "Your listing has been approved!"
   - Queues email to owner with approval + premium benefits ad
   - Returns updated listing

4. /app/api/admin/listings/[id]/reject/route.ts:
   - POST: Reject a pending listing
   - Body: { reason: string }
   - Sets status = 'rejected'
   - Sets rejection_reason = reason
   - Creates notification for owner with reason
   - Queues email to owner with rejection details
   - Returns updated listing

5. /app/api/admin/listings/bulk/route.ts:
   - POST: Bulk actions on multiple listings
   - Body: { listing_ids: string[], action: "approve"|"reject"|"activate"|"deactivate"|"delete", reason?: string }
   - Processes each listing
   - Creates notifications for each affected owner
   - Returns: { success_count, failed_count, errors: [] }

6. /app/api/admin/listings/[id]/notes/route.ts:
   - GET: Fetch admin notes for a listing
   - POST: Add admin note
   - Body: { note: string }
   - Returns: note with admin name and timestamp

7. /app/api/admin/listings/[id]/assign-owner/route.ts:
   - POST: Assign a business owner to a listing
   - Body: { owner_id: string }
   - Updates listing.owner_id
   - Sets is_pre_populated = false
   - Creates notification for new owner
   - Queues welcome email

8. /app/api/admin/listings/[id]/analytics/route.ts:
   - GET: Detailed analytics for a listing
   - Query params: ?period=7d|30d|90d|all
   - Returns:
     {
       total_views: number,
       total_clicks: { phone, email, website, directions, social, share },
       daily_views: [{ date, count }],
       daily_clicks: [{ date, count }],
       top_referrers: [{ source, count }] (if tracked)
     }

9. /app/api/admin/listings/counts/route.ts:
   - GET: Quick counts for sidebar badges and status tabs
   - Returns: { pending, approved, rejected, draft, claimed_pending, total, active, inactive }
